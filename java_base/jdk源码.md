## 设计模式
1. 设计模式6大原则
   - 单一原则(一个类和方法只做一件事)
   - 里氏替代(多态,子类扩展父类)
   - 依赖倒置(细节依赖抽象,下层依赖上层)
   - 接口隔离(建立单一接口)
   - 迪米特原则(最少知道,降低耦合)
   - 开闭原则(抽象架构,扩展实现)
2. 创建型模型:这种模式提供了创建对象的机制,能够提升自己已有代码的灵活性和可复用性
3. 结构型模型:这类模式介绍如何将对象和类组装成较大的结构,并同时保持结构的灵活和高校
4. 行为模式:这类模式负责对象将的高效沟通和职责委派
